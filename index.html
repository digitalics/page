JavaScript is a programming language that is used on websites to make them interactive and optimized. The language was first made in 1995 and even though the name clearly contains the word "Java" it in actuality was little to nothing to do with Java which is a different programming language. JavaScript went from being a nobody to being a somebody and it has seen a massive rise in popularity these recent years, so much so that it is the most taught programming language in the world wide web.

Well, JavaScript seems all fine and dandy but why should I choose JS and not Python or C#?

JS has the advantage of being an easy applicable language that gives instant feedback to the developer, you can easily see where your skills are lacking and what you need to learn to make things work. Python has a lot of different uses from machine learning to data-analytics(you probably don't even know what a data analyst does) and it can quickly become very overhelming when you're about to choose a project or task to improve your skills. C# is a strong contender that is sought after in todays marketplace, it too has many uses but I would still advise you to choose JavaScript because C# is Microsoft specific.

Benefits:
  • Good beginner language
  • High demand
  • Easy debugging(finding bugs in the code)
  • And many more...

Detriments:
  • Imperative to be meticulous with the security(even Google struggles with this)
  • Limited to the web
  • Difficult to master

Now when moving around and creating projects you might stumble upon terms like OOP, DOM, AJAX, API, framework and the "this" keyword. My recommendation is to only focus on the DOM(Document Object Model) or in other words "the art of making a website cool with your JS". DOM helps you to see where the JavaScript is suppose to be applied, when learning DOM you will naturaly see where the other stuff such AJAX and frameworks comes into play.
AJAX, different frameworks and different API:s are definantly going to raise your pay, but they are more like specialization and less like mandatory knowledge for developers.

So when you've coded for a while you will start seeing the words RegEx and hashing being thrown around. RegEx is a technology that would help every developer in every field but it's not necessary to know. Dynamic programing and hashing are ways of structuring algorithms to make them more efficient, whilst you can get away with not learning DP trying to escaping hashing won't be easy since alot of interview question involve hashing and DP.
